# RSS 扩展中文说明

RSS 是一个面向 Flarum 2.x 的 RSS/Atom 聚合扩展。它可以抓取外部独立博客或站点的文章，在论坛内提供独立 RSS 页面，也可以把 RSS 文章按发布时间混入首页帖子列表。

这个扩展的核心设计是：RSS 文章不会一抓取就写入 Flarum 的帖子表。只有当用户真的要围绕某篇 RSS 文章评论时，扩展才会把这篇文章转换成真实的 Flarum 讨论。

## 功能介绍

- 在后台管理 RSS/Atom 站点。
- 支持用户前台提交 RSS 源，默认状态为待审核。
- 管理员可以审核、删除、编辑 RSS 源。
- 后台 RSS 源列表支持绑定用户头像。
- 每个 RSS 源可以绑定一个 Flarum 用户；RSS 文章生成真实讨论后，会使用这个用户作为讨论作者。
- 后台可以实时查看某个 RSS 源当前返回的文章列表。
- 后台列表底部保留一个空表单，方便连续添加新的 RSS 源。
- 支持独立的“独立博客”页面展示 RSS 文章。
- 支持把 RSS 文章混入首页主帖子列表。
- 首页工具栏支持在“RSS 综合列表”和“原生帖子列表”之间临时切换。
- 首页工具栏支持直接打开 RSS 源提交弹窗。
- 个人设置页新增独立的“RSS”设置分组，用户可以决定首页是否默认展示 RSS 文章。
- RSS 文章列表使用类似原生帖子的 UI。
- RSS 文章的评论图标和评论数量可以进入讨论页。
- 未评论前显示虚拟讨论页，不会创建 Flarum 讨论数据。
- 首次评论后才创建真实 Flarum 讨论和评论。
- 生成真实讨论时可以自动挂载后台配置的标签。
- 内置中文和英文语言包。

## 环境要求

- Flarum `^2.0.0-beta`
- PHP `^8.3`
- Composer

## 安装

使用 Composer 安装：

```sh
composer require shebaoting/rss:"^2.0"
php flarum migrate
php flarum cache:clear
```

然后进入 Flarum 后台启用扩展。

## 更新

```sh
composer update shebaoting/rss:"^2.0"
php flarum migrate
php flarum cache:clear
```

## 后台设置

进入后台的 RSS 扩展页面，可以配置：

- **在首页主列表显示 RSS 文章**：开启后，RSS 文章可以混入首页帖子列表。
- **RSS 评论生成帖子时使用的标签**：用户首次评论 RSS 文章并创建真实讨论时，讨论会挂载到这个标签。

RSS 站点列表支持：

- 编辑 RSS 站点名称。
- 编辑 RSS 站点链接。
- 绑定、修改或删除关联用户。
- 切换审核状态。
- 查看该站点实时 RSS 文章列表。
- 删除站点。
- 在列表底部使用空表单继续添加新站点。

用户在前台提交的 RSS 源默认是待审核状态，管理员审核通过后，该 RSS 源的文章才会进入公开展示。

## 前台使用

如果后台开启了“在首页主列表显示 RSS 文章”，首页工具栏会出现两个 RSS 相关按钮：

- 切换按钮：在 RSS 综合列表和原生帖子列表之间切换。
- 添加按钮：打开 RSS 源提交弹窗。

用户也可以进入 `/settings`，在独立的 **RSS** 设置分组里控制自己的默认首页模式：

- 开启“首页展示 RSS 文章”：以后打开首页默认显示 RSS 综合列表。
- 关闭“首页展示 RSS 文章”：以后打开首页默认显示原生帖子列表。

这个个人设置只影响用户自己的默认首页模式，不会影响其他用户。

## RSS 文章和评论机制

RSS 文章抓取后会保存在扩展自己的 `rss_items` 表中，不会直接写入 Flarum 的 `discussions` 表。

在没有用户评论之前：

- RSS 文章会在列表里模拟原生帖子的样式展示。
- 点击文章标题会打开原始来源网址。
- 点击评论图标或评论数量会进入虚拟 RSS 讨论页。
- 虚拟讨论页只显示原文网址作为正文内容。
- 数据库里还没有真实的 Flarum 讨论。

当用户首次评论时：

- 扩展创建真实的 Flarum 讨论。
- 扩展创建第一条内容为原文网址的首帖。
- 扩展把用户评论写入真实评论帖。
- 讨论作者使用该 RSS 源绑定的用户。
- 后台设置的标签会被挂载到这个讨论。
- RSS 文章会记录关联的 `discussion_id`，之后再次点击评论会直接进入真实讨论。

这种方式可以避免大量外部文章直接堆进 Flarum 帖子表，同时又保留社区围绕 RSS 文章展开讨论的能力。

## 抓取 RSS

扩展注册了 `rss:fetch` 命令。

抓取所有已审核 RSS 源：

```sh
php flarum rss:fetch
```

测试单个 RSS 地址：

```sh
php flarum rss:fetch "https://example.com/feed.xml"
```

扩展会通过 Flarum 调度器每小时执行一次抓取。请确保服务器已经配置 Flarum 定时任务，例如：

```sh
* * * * * cd /path/to/flarum && php flarum schedule:run >> /dev/null 2>&1
```

## 从旧版本升级

`2.0.0` 之后的版本用于 Flarum 2.x。如果你之前在 Flarum 1.x 使用过这个扩展，可以保留旧数据库数据，然后执行：

```sh
composer require shebaoting/rss:"^2.0"
php flarum migrate
php flarum cache:clear
```

扩展会继续使用已有的 RSS 源和 RSS 文章数据，并通过迁移补齐 Flarum 2.x 所需字段。

## 常见说明

- 如果 RSS 文章没有出现在首页，请先确认 RSS 源已审核。
- 如果首页没有混入 RSS 文章，请检查后台“在首页主列表显示 RSS 文章”是否开启。
- 如果某个用户不想默认看到 RSS 综合列表，可以在个人设置的 RSS 分组里关闭“首页展示 RSS 文章”。
- 如果评论后生成的讨论作者不正确，请检查该 RSS 源是否绑定了正确的用户。
- 如果生成的讨论没有标签，请检查后台“RSS 评论生成帖子时使用的标签”是否已设置。
